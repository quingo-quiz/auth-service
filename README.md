# Quingo Auth Service

Микросервис аутентификации и авторизации для платформы Quingo.

## Основные возможности и фичи

*   **Гибкая аутентификация**:
    *   Классический вход через `email` + `password`.
    *   **2FA (TOTP)**: Поддержка двухфакторной аутентификации с помощью одноразовых кодов.
    *   **OAuth2/OIDC**: Интеграция с Google и GitHub для бесшовной аутентификации.
    * Поддержка безопасного слияния аккаунтов при использовании различных способов входа.
*   **Управление сессиями**:
    *   **Двухуровневая модель JWT токенов**: Access с коротким TTL и долгоживущий Refresh для механизма ротации.
    *   **White-list**: Отслеживание активных access токенов в Redis с возможность мгновенного отзыва.
*   **Интеграция с Kafka**: Асинхронная публикация событий в топик `notifications-events` для последующей обработки микросервисом уведомлений.
*   **Outbox Pattern**:
    *   Внешние события сначала регистрируются в БД, что позволяет не потерять их даже при падении сервиса.
    *   Фоновый **Outbox Scheduler** гарантирует доставку сообщения в Kafka даже при временных сбоях сети.
*   **API-First**: Контракты описаны с помощью **TypeSpec**, что позволяет генерировать актуальную документацию.

## 🏗 Инфраструктура и Деплой

Сервис полностью контейнеризирован и адаптирован для работы в облачных средах:
*   **Kubernetes (k3s)**: Оркестрация контейнеров в легковесном кластере.
*   **Ingress Nginx**: Управление внешним трафиком и маршрутизация.
*   **Kafdrop**: Интегрированный Web-интерфейс для мониторинга топиков и сообщений в Kafka.
*   **Swagger UI**: Сервис интерактивной документации. Собирает документацию api с каждого микросервиса платформы.

### CI/CD и автоматизация
В проекте настроены пайплайны:
*   **Автоматическое обновление документации**: При изменении TypeSpec-моделей новая OpenAPI спецификация генерируется и деплоится автоматически, обеспечивая актуальность.
*   **Основной пайплайн**: 
     * **Тестирование:** Прогоняются unit и интеграционные тесты по кодовой базе.
    * **Сборка имаджа:** При успешном прохождении тестов собирается образ приложения и пушится на докерхаб.
    * **Деплой:** После успешной сборки имадж деплоймента в k3s кластере обновляется на новую версию.


### Тестирование
Для обеспечения стабильности используется многоуровневое тестирование:
*   **Unit-тесты**: Покрытие бизнес-логики сервисов с использованием JUnit 5 и Mockito.
*   **Integration-тесты**: Тестирование взаимодействия с реальными компонентами (PostgreSQL, Redis, Kafka) с помощью **Testcontainers**, что гарантирует корректную работу в продакшн среде.

## 💻 Quick Start

1.  **Клонируйте репозиторий**:
    ```bash
    git clone https://github.com/quingo-quiz/auth-service.git
    cd auth-service
    ```
2.  **Создайте файл `.env`** в корне проекта на основе примера ниже.

#### Пример `.env` файла
```properties
# App Config
APP_DOMAIN=localhost
SPRING_PROFILES_ACTIVE=local
FRONTEND_URL=http://localhost:3000/

# PostgreSQL
POSTGRES_HOST=localhost
POSTGRES_USER=admin
POSTGRES_PASSWORD=admin
POSTGRES_DB=auth-service-db

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=admin
REDIS_MAXMEMORY=256mb

# Kafka
KAFKA_BOOTSTRAP_SERVERS=localhost:9092

# JWT
JWT_ISSUER=localhost
JWT_SECRET=your_very_long_and_secure_secret_string_here
JWT_ACCESS_MINUTES=15
JWT_REFRESH_DAYS=30

# OAuth2
GOOGLE_CLIENT_ID=you_can_skip_this
GOOGLE_CLIENT_SECRET=you_can_skip_this
GITHUB_CLIENT_ID=you_can_skip_this
GITHUB_CLIENT_SECRET=you_can_skip_this

# For safely storing data
ENCRYPTION_KEY=461ffkroeprogtla83asbr230dsd2lvw
```
3.  **Запустите приложение**:
    ```bash
    docker-compose up -d --build
    ```

4. **Теперь приложение доступно по http://localhost:8080/api/auth**
